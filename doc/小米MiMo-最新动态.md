## 3. 对这个项目的实际判断

### 3.1 第一阶段选什么模型

建议：

- 默认：`mimo-v2.5-pro`
- 成本敏感时：`mimo-v2.5`

### 3.2 第一阶段选什么接法

建议：

- 默认：OpenAI 兼容 API
- 默认：按量付费 `payg`

原因：

- 最容易跑通
- SDK 成熟
- 调试成本低

### 3.3 什么时候切 Token Plan

当你开始出现这些情况时再切：

- 多人共享调用
- 每天持续生成培训问答与题目
- 需要稳定预算

## 4. 一个关键兼容性提醒

官方 Codex 文档明确提到：

- MiMo **暂不兼容 Responses API**
- 目前更适合使用 **Chat Completions API**

这意味着如果你后面用的是只支持 `responses` 的新式接法，要额外处理，不要直接套。
